# Color Adjacency Bonus — PPO Run 6 Design

**Date:** 2026-02-26
**Status:** Implemented
**Files:** `python/game/env.py`, `python/tests/test_env_rewards.py`

---

## Problem: Sparse Signal → Critic Ceiling at 0.22 EV

PPO Run 5 peaked at eval=5.52 @ 500k steps then degraded. Tensorboard shows
`explained_variance` capped at ~0.22 throughout training.

Root cause: the critic sees mostly `survival_bonus (0.05) + tiny height_delta`
on every step. The two signals that correlate with board quality —
`squares_delta` and `chain_delta` — only fire when a 2×2 pattern is completed
or the chain length changes. In a 16-column board with random play, these fire
on ~20–30% of steps early in training, giving the critic little to learn from
on the majority of steps.

With such sparse signal the critic cannot distinguish "building toward a good
state" from "building toward a bad state". Explained variance stays low, the
value baseline is noisy, and PPO's advantage estimates are unreliable — policy
gradient steps become high-variance and learning stalls.

---

## Approach: Same-Color Adjacency Contacts

Add a small per-step bonus for placing block cells adjacent to same-color
existing board cells. This fires on every placement, even before any 2×2
pattern is complete.

### Geometry

For a 2×2 block placed at `(drop_x, drop_y)`:

```
             col:  x-1   x    x+1   x+2
row drop_y-1:      [^L]  [TL] [TR]  [^R]   ← above neighbors (V-above)
row drop_y:        [<L]  [TL] [TR]  [>R]   ← horizontal neighbors
row drop_y+1:      [<R]  [BL] [BR]  [>R]   ← horizontal neighbors
```

Contacts counted:
- **Horizontal (4 max):** left neighbor of TL/BL, right neighbor of TR/BR
- **Vertical-above (2 max):** cell above TL, cell above TR
- **Total max: 6 contacts**

Cells below the block are excluded — they will be checked when the *next*
block lands on top, so there is no double-counting incentive.

### Weight

```python
ADJ_BONUS_WEIGHT = 0.04
max_adj_bonus = 6 × 0.04 = 0.24
```

Chosen so that:
1. A step with max 6 contacts (0.24) is roughly half the `survival_bonus`
   (0.05 × step_count relationship), keeping it secondary to actual scoring.
2. Magnitude is small enough that it does not overwhelm `squares_delta` or
   `chain_delta` signals.
3. It fires on every placement, giving the critic a non-zero signal to fit
   against even on steps where no patterns form.

---

## Implementation

### Timing

`hard_drop()` replaces `current_block` with the next block internally. To
access the block that was actually placed, we capture before `hard_drop()`:

```python
pre_drop_y = find_drop_position(board, current_block, actual_x, block_y, falling_cols)
pre_drop_pattern = [row[:] for row in current_block.pattern]
```

After `hard_drop()` + timeline ticks + gravity settle, the neighbor cells at
`actual_x-1` and `actual_x+2` are pre-existing (outside the 2×2 footprint),
so reading them from the post-placement board is valid.

### Zero on game over

`adj_bonus = 0.0` when `status == "gameOver"` to avoid rewarding the terminal
step.

### Full reward formula (PPO Run 6)

```python
reward = score_delta
       + squares_delta * 0.2
       + chain_delta   * 0.3
       + height_delta
       + adj_bonus              # NEW
       + survival_bonus
       + death_penalty
```

## Expected Impact

| Metric | Before (Run 5) | Expected (Run 6) |
|--------|---------------|-----------------|
| `explained_variance` | ~0.22 (ceiling) | > 0.35 |
| `adj_bonus` during play | n/a | > 0 on most mid/late steps |
| Peak eval score | 5.52 @ 500k | higher + more stable |

The denser per-step signal should help the critic converge faster and maintain
higher explained variance, leading to more reliable advantage estimates and
sustained policy improvement beyond 500k steps.
